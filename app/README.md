# Termex — Flutter App

This is the **in-progress Flutter rewrite** of Termex. The production version still ships on the Tauri/Vue stack (see [../README.md](../README.md)). This directory will become the default at v0.49.0 release.

**Status**: Migration foundation complete, UI layouts done, FRB bindings pending codegen, 71 TODOs open. See [../docs/iterations/v0.51.0-remediation.md](../docs/iterations/v0.51.0-remediation.md).

---

## Prerequisites

- **Rust** (stable, via [rustup](https://rustup.rs/))
- **Flutter SDK** 3.24+ ([install guide](https://docs.flutter.dev/get-started/install))
- **flutter_rust_bridge codegen**:
  ```bash
  cargo install flutter_rust_bridge_codegen --version '^2.0'
  ```

Verify your setup:
```bash
flutter --version           # should show 3.24.0+
flutter doctor              # address any red items
flutter_rust_bridge_codegen --version   # should show 2.x
```

---

## Setup (first time)

```bash
cd ..                              # repo root
./scripts/frb-codegen.sh           # generate Dart bindings from crates/termex-flutter-bridge/src/api/
cd app
flutter pub get                    # resolve Dart dependencies
```

Now the Flutter app can actually call the Rust backend. Without running codegen, the app compiles but every bridge call throws `UnimplementedError`.

## Run

```bash
flutter run -d macos               # macOS
flutter run -d windows             # Windows
flutter run -d linux               # Linux
```

Hot-reload is fully supported for Dart. Changes to Rust require a rebuild:
```bash
cargo build --release -p termex-flutter-bridge
flutter run -d macos               # or restart running instance with `R`
```

### Regenerating FRB bindings

Whenever you change `crates/termex-flutter-bridge/src/api/*.rs`:

```bash
cd ..                         # repo root
./scripts/frb-codegen.sh      # regenerate
git add app/lib/src/ crates/termex-flutter-bridge/include/
git commit -m "chore(frb): regenerate bindings"
```

CI will reject PRs where bindings are out of sync.

### State management

This app uses **Riverpod** (not `provider`, not `flutter_bloc`). Conventions:

- Each feature module has a `state/` directory with providers:
  ```
  features/server_list/state/
  ├── server_provider.dart         # List + CRUD
  ├── session_provider.dart        # Active sessions
  └── quick_connect_provider.dart  # Quick connect history
  ```
- Providers expose async notifiers that wrap FRB calls.
- Never call FRB directly from widgets — always through a provider.

## Anti-reverse-engineering (sentinel) builds

Release builds include optional integrity checks gated by the `SENTINEL` dart-define:

```bash
# Normal release (sentinel checks tree-shaken out)
flutter build macos --release

# With sentinel checks
flutter build macos --release --dart-define=SENTINEL=true
```

Rust-side equivalent uses the `sentinel` feature:
```bash
cargo build -p termex-core --features sentinel --release
```

See `lib/system/sentinel_flag.dart` and `../CLAUDE.local.md` for details.
Watermarks are recorded in `../watermarks.local.json` (gitignored).

## Troubleshooting

### "FRB stub hit" error at runtime

Means the generated bindings are missing. Run:
```bash
./scripts/frb-codegen.sh
```

### Dart analyzer complains about missing types

The analyzer runs against `lib/src/frb_generated.dart`. If codegen hasn't run, the analyzer sees stubs. Run codegen first, then `flutter pub get`.

### Rust side changes not reflected

Flutter hot-reload only touches Dart. After changing Rust:
```bash
cargo build -p termex-flutter-bridge --release
# Restart the running flutter instance (press Shift+R in terminal)
```

### `flutter doctor` reports missing platform toolchain

- **macOS**: install Xcode
- **Windows**: Visual Studio 2022 with "Desktop development with C++" workload
- **Linux**: `sudo apt install clang cmake ninja-build pkg-config libgtk-3-dev liblzma-dev libstdc++-12-dev`
