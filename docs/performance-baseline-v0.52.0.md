- Platform: macOS arm64 (Darwin 24.6.0)
- Dart: 3.11.5 / Flutter: 3.41.7

## Notes

- Timings are **debug build** — a release build is typically 2–5× faster
- All targets met with wide headroom; no release-gate performance concerns
- Terminal frame budget (60 fps → 16 ms/frame) is honoured by all hot paths
- **Not yet measured** (requires integration_test harness on real windows):
  - App cold-start time (target < 2 s)
  - Keystroke latency (target < 16 ms end-to-end)
  - 20-tab steady memory (target < 100 MB)

Those measurements are tracked in [v0.52.0 §C5](iterations/v0.52.0-gap-coverage.md#c5-性能基准达标复核); a follow-up runs them under the `integration_test/performance/` harness once Flutter driver is wired up.

## Methodology

Each scenario runs a self-contained synthetic load and stops the stopwatch before assertions. The numbers reflect pure Dart CPU cost — no GPU/frame budget is simulated. Real-world performance must be verified with `flutter run --release --profile` and DevTools on a populated workspace.
