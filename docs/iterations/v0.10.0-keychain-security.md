# v0.10.0 -- OS Keychain Security

> Status: **Planned**
> Depends on: v0.9.0

## Goals

将所有敏感凭证（SSH 密码、私钥 passphrase、AI API Key）迁移至操作系统原生密钥管理器，实现零摩擦安全存储。`termex.db` 不再存储任何明文或加密密码，仅保留 keychain 映射 ID。

## Background

当前安全问题：
- `termex.db` 文件泄露 = 所有密码/密钥泄露
- 无主密码时，`password_enc` 字段存储明文字节
- 用户每次启动输入主密码的体验不友好

## Architecture

```
termex.db (可公开)              OS Keychain (系统保护)
┌─────────────────────┐        ┌──────────────────────┐
│ servers:             │        │                      │
│   password_enc: NULL │───────▶│ termex:ssh:{uuid}    │
│   keychain_id: "xxx" │        │  → "real_password"   │
│                      │        │                      │
│ ai_providers:        │        │ termex:ai:{uuid}     │
│   api_key_enc: NULL  │───────▶│  → "sk-xxxxx"        │
│   keychain_id: "yyy" │        │                      │
└─────────────────────┘        └──────────────────────┘
```

### 平台后端

| 平台 | Keychain 后端 | 安全级别 |
|------|-------------|---------|
| macOS | Keychain Services (Secure Enclave) | 硬件级，支持 Touch ID |
| Windows | Credential Manager (DPAPI) | 用户登录密码保护 |
| Linux | Secret Service (GNOME Keyring / KDE Wallet) | 用户登录密码保护 |

### Fallback 策略

```
启动 → 检测 OS Keychain 可用性
         │
         ├─ 可用 → 使用 Keychain（零摩擦，推荐）
         │
         └─ 不可用（headless Linux 等）
              → Fallback 到 AES-256-GCM + 主密码
              → 提示用户设置主密码
```

## Features

### Rust Keychain 模块
- [ ] 新增 `src-tauri/src/keychain/` 模块
- [ ] 依赖 `keyring = "3"` crate，统一封装三平台 API
- [ ] API: `store(service, key, value)`, `get(service, key)`, `delete(service, key)`
- [ ] Keychain 可用性检测 `is_available() -> bool`
- [ ] 错误处理：keychain 锁定、权限不足、不可用等

### SSH 密码迁移
- [ ] `servers` 表新增 `keychain_id` 字段（TEXT, nullable）
- [ ] 创建/编辑服务器时：密码写入 keychain，`keychain_id` 存入 db，`password_enc` 置 NULL
- [ ] SSH 连接时：通过 `keychain_id` 从 keychain 取密码
- [ ] 编辑回填：从 keychain 读取密码显示
- [ ] 删除服务器时：同步删除 keychain 条目
- [ ] Passphrase 同理

### AI API Key 迁移
- [ ] `ai_providers` 表新增 `keychain_id` 字段
- [ ] 保存 provider 时：API Key 写入 keychain
- [ ] AI 调用时：从 keychain 读取 API Key
- [ ] 编辑回填：从 keychain 读取
- [ ] 删除 provider 时：清理 keychain

### 数据迁移（V3 Migration）
- [ ] 检测现有 `password_enc` / `api_key_enc` 非 NULL 的记录
- [ ] 自动迁移到 keychain，更新 `keychain_id`，清空 `*_enc` 字段
- [ ] 迁移失败时保留原有加密字段，不丢数据

### 配置导出/导入适配
- [ ] 导出时：从 keychain 读取密码，用导出密码加密后写入 `.termex` 文件
- [ ] 导入时：从 `.termex` 文件解密密码，写入 keychain

### Fallback 主密码模式
- [ ] Keychain 不可用时，提示用户设置主密码
- [ ] 使用现有 AES-256-GCM + Argon2id 方案
- [ ] 主密码存储在内存中，锁定时清零
- [ ] 设置面板中显示当前安全模式（Keychain / 主密码）

### 安全设置面板
- [ ] 显示当前凭证保护方式（OS Keychain / Master Password）
- [ ] 显示 keychain 中存储的凭证数量
- [ ] 主密码设置/修改（fallback 模式）
- [ ] 一键清除所有 keychain 凭证

## Keychain ID 命名规范

```
termex:ssh:password:{server_uuid}       # SSH 密码
termex:ssh:passphrase:{server_uuid}     # SSH 私钥 passphrase
termex:ai:apikey:{provider_uuid}        # AI API Key
termex:master:verify                     # 主密码验证令牌（fallback）
```

## Security Considerations

- Keychain 条目绑定应用签名（macOS），未签名应用每次访问会弹授权
- Windows DPAPI 绑定用户登录凭证，重装系统后无法恢复
- 建议用户定期导出配置备份
- keychain 中的密码不记录到日志
