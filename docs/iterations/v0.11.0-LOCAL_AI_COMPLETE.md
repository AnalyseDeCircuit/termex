## 概述

### 什么是本地 AI 模型？

Termex v0.11.0 在现有云端 AI 提供商（OpenAI、Claude、Gemini）基础上，新增两类本地 AI 模型支持，实现完全离线的 AI 能力：

**Type 1：局域网 Ollama**（扩展现有）
- 用户在局域网服务器部署 Ollama
- Termex 通过自定义 URL 连接（如 `http://192.168.1.100:11434`）
- UI 显示 🌐 LAN 徽章区分

**Type 2：本机离线模型**（完全新增）
- 内置 llama-server 推理引擎二进制
- 模型文件热下载到 `~/.termex/models/`
- 完全离线运行，无需网络和 API Key
- 12 个代表性模型，4 层级分类（micro/small/medium/large）

### 核心价值

✅ **零成本**：无需购买 API 额度，不依赖云端服务
✅ **隐私**：数据完全本地存储，不上传到任何云端
✅ **离线**：断网后 AI 功能仍然可用
✅ **灵活**：支持多种开源模型，可按硬件选择
✅ **一键使用**：自动检测、下载、启动，无需手动配置

### 本机模型三梯度分类

| 梯度 | 大小 | RAM 要求 | 速度 | 质量 | 推荐模型 |
|------|------|---------|------|------|---------|
| **Micro** | < 200MB | 4GB | 极快 | 基础 | TinyLlama 1.1B |
| **Small** | 200M-1GB | 8GB | 快速 | 良好 | Qwen2.5 0.5B / Phi-3 3B |
| **Medium** | 1GB-5GB | 8GB+ | 中等 | 优秀 | Llama3.2 3B / Qwen2.5 3B |
| **Large** | 5GB+ | 16GB+ | 较慢 | 最优 | **Qwen2.5 7B ⭐**（推荐） |

### 数据流

```
用户操作              Termex                    系统
  │                    │                         │
  ├─ "下载模型"        │                         │
  │                    ├─ 检查磁盘空间 ─────────→ ~/.termex/
  │                    │ ✓ 空间足够
  │                    │                         │
  │◄──────────────────┤ 显示进度条               │
  │ HTTP Range Request │──────────────────────────→ HuggingFace
  │                    │◄────────────────────────── 流式下载
  │ emit progress      │ emit local-ai://download
  │ event (1x/sec)     │ DownloadProgress
  │                    │                         │
  │ 下载完成            │ 计算 SHA256
  │                    │ ✓ 验证通过
  │                    │ 移动到 ~/.termex/models/
  │                    │                         │
  ├─ "启用为 AI"       │                         │
  │                    ├─ 创建 provider record
  │                    ├─ 启动 llama-server
  │                    │  ├─ 找到空闲端口
  │                    │  ├─ spawn 子进程
  │                    │  ├─ 加载模型文件
  │                    │  └─ 监听 localhost:{port}
  │                    │                         │
  │ 设为默认 AI        │                         │
  │ 输入命令提示        │                         │
  │                    ├─ 调用 ai_provider
  │◄──────────────────┤ GET localhost:{port}
  │ "ls 列出文件..."   │ /v1/chat/completions
  │                    │                         │
```

## 快速开始

### 开发者环境

#### 1. 安装 llama-cpp

```bash
# macOS
brew install llama.cpp

# Linux (Ubuntu/Debian)
sudo apt-get install llama-cpp

# Linux (Fedora/RHEL)
sudo dnf install llama-cpp

# Windows（通过 Scoop 或 Chocolatey）
scoop install llama-cpp
# 或
choco install llama-cpp
```

验证安装：
```bash
which llama-server
llama-server --help
```

#### 2. 运行开发环境

```bash
cd termex
pnpm install
pnpm tauri dev
```

#### 3. 访问本地 AI 功能

1. 打开应用设置
2. 进入 **AI 配置**
3. 查看 **本机离线模型** 面板（顶部）
4. 选择模型并点击 **下载**
5. 下载完成后点击 **用作 AI 提供商**
6. 在 AI 面板测试（"explain ls"）

## 故障排除

### 常见问题

**Q1: llama-server 找不到**

```bash
# 检查安装
which llama-server
llama-server --help

# 如果没有，安装 llama.cpp
brew install llama.cpp  # macOS
sudo apt-get install llama-cpp  # Linux

# 自定义路径
export LLAMA_SERVER_PATH=/path/to/llama-server
```

**Q2: SHA256 验证失败**

- 源文件被篡改或损坏
- 更新 local-models.json 中的 SHA256 值
- 手动验证：`sha256sum ~/.termex/models/*.gguf`

**Q3: 推理响应超时**

```bash
# 直接测试 API
curl http://localhost:15000/v1/models | jq

# 检查模型是否加载
curl http://localhost:15000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"default","messages":[{"role":"user","content":"Hi"}]}'
```

**Q4: 内存不足**

- 选择更小的模型（Micro/Small tier）
- 检查硬件规格（至少 4GB RAM）
- 监控进程内存占用

**Q5: 网络中断下载**

- 下载会自动断点续传
- 网络恢复后点击继续下载
- 前提：已下载部分保留在 ~/.termex/models/

ps aux | grep llama-server

## 性能优化

### 关键路径分析

```
用户点击下载 (0ms)
  ├─ 磁盘检查 (100ms)
  ├─ HTTP Range 请求 (网络延迟)
  └─ SHA256 验证 (CPU 密集，~10-20s)
     └─ 文件移动 (10ms)

用户点击"用作 AI" (0ms)
  ├─ 查找空闲端口 (~10ms)
  ├─ spawn 子进程 (~50ms)
  ├─ 加载模型文件 (取决于大小，~500ms-2s)
  └─ API 就绪 (通过轮询检测)

AI 功能调用 (0ms)
  ├─ HTTP 请求 (网络延迟)
  ├─ llama 推理 (取决于硬件，~2-30s)
  └─ 流式响应 (实时)
```

### 优化建议（v0.11.1+）

1. **下载防抖**：减少进度事件发送频率（100ms 节流）
2. **SHA256 异步**：分块计算，不阻塞 UI
3. **并行下载**：允许 3 个并发下载
4. **引擎预热**：用户下载时预启动引擎
5. **GPU 加速检测**：自动启用 CUDA/Metal/Vulkan
