# V0.16.0 — 自定义快捷键

## Context

当前 Termex 的快捷键（Cmd+N / Cmd+W / Cmd+Tab 等）硬编码在 `useShortcuts.ts` 中，Settings → Keybindings 页面仅展示快捷键列表，用户无法自定义。

本迭代目标：
1. 允许用户自定义所有快捷键
2. Keybindings 面板中点击快捷键标签进入录入模式，按下组合键即完成设置
3. 保存后立即生效，无需重启

---

## 一、当前状态分析

### 现有快捷键（硬编码于 `useShortcuts.ts`）

| 动作 ID | 默认快捷键 | 说明 |
|---------|-----------|------|
| `newConnection` | `Cmd+N` / `Ctrl+N` | 新建连接 |
| `openSettings` | `Cmd+,` / `Ctrl+,` | 打开设置 |
| `toggleSidebar` | `Cmd+\` / `Ctrl+\` | 切换侧边栏 |
| `closeTab` | `Cmd+W` / `Ctrl+W` | 关闭当前标签 |

| `goToTab` | `Cmd+1~9` / `Ctrl+1~9` | 跳转到第 N 个标签 |

### 现有 UI（`KeybindingsTab.vue`）

纯展示列表，`<kbd>` 标签显示快捷键文本，无交互能力。

## 二、快捷键数据模型

### 类型定义 (`src/types/keybindings.ts`)

```typescript
/** All bindable action identifiers. */
export type KeybindingAction =
  | "newConnection"
  | "openSettings"
  | "toggleSidebar"
  | "toggleAi"
  | "closeTab"
  | "nextTab"
  | "prevTab"
  | "goToTab1"
  | "goToTab2"
  | "goToTab3"
  | "goToTab4"
  | "goToTab5"
  | "goToTab6"
  | "goToTab7"
  | "goToTab8"
  | "goToTab9";

/** A keyboard shortcut composed of modifier keys + a main key. */
export interface Keybinding {
  /** Modifier keys. Platform-adaptive: metaKey on macOS, ctrlKey on others. */
  mod: boolean;
  shift: boolean;
  alt: boolean;
  /** The primary key (KeyboardEvent.key value, e.g., "n", "Tab", "1"). */
  key: string;
}

/** Map from action ID to its keybinding. */
export type KeybindingMap = Record<KeybindingAction, Keybinding>;
```

### Keybindings 设置面板改造

```
│  │ Open Settings        │  [ ⌘ ,           ] [↺]        │ │
│  │ Toggle Sidebar       │  [ ⌘ \           ] [↺]        │ │
│  │ Toggle AI Panel      │  [ ⌘ ⇧ I         ] [↺]        │ │
│  │ Close Current Tab    │  [ ⌘ W           ] [↺]        │ │
│  │ Next Tab             │  [ ⌘ Tab         ] [↺]        │ │
│  │ Previous Tab         │  [ ⌘ ⇧ Tab       ] [↺]        │ │
│  │ Go to Tab 1~9        │  [ ⌘ 1~9        ] [↺]        │ │
│  └──────────────────────┴────────────────────────────────┘ │
└────────────────────────────────────────────────────────────┘

点击快捷键标签后进入录入模式：

│  New Connection       │  [ ⏳ Press shortcut...  ] [✕]   │
```

### UI 元素

- **动作名称**：左侧显示 i18n 翻译的动作描述
- **快捷键标签**：右侧可点击的 `<kbd>` 区域，显示当前快捷键
  - 普通状态：显示格式化的快捷键文本（如 `⌘ N`）
  - 录入状态：显示 "Press shortcut..." 提示文本，边框高亮
- **重置按钮** `[↺]`：恢复该项到默认快捷键（仅当与默认值不同时显示）
- **全局重置** `[Reset All]`：右上角按钮，恢复所有快捷键到默认值

### 录入交互流程

1. 用户点击某行的快捷键标签 → 进入录入模式
2. 标签变为 "Press shortcut..." + 高亮边框 + 脉冲动画
3. 用户按下组合键（如 `Cmd+Shift+K`）→ 捕获按键事件
4. **冲突检测**：检查是否与其他 action 已绑定的快捷键冲突
   - 无冲突 → 保存并退出录入模式，显示新快捷键
   - 有冲突 → 提示 "Already used by {action name}"，不保存，保持录入模式
5. 按 `Escape` → 取消录入，恢复原快捷键

### 录入规则

- **必须包含修饰键**：纯字母/数字键不允许作为快捷键（避免与终端输入冲突）
- **合法修饰键组合**：`Cmd/Ctrl` + 可选 `Shift` + 可选 `Alt` + 主键
- **主键限制**：字母 a-z、数字 0-9、符号键（`,` `.` `/` `\` `[` `]` `-` `=`）、功能键（F1-F12）、特殊键（Tab、Enter、Backspace、Delete、Space、方向键）
- **禁止绑定的组合**：`Cmd+C`（复制）、`Cmd+V`（粘贴）、`Cmd+A`（全选）、`Cmd+Q`（退出）— 系统保留

### 快捷键显示格式

macOS 使用符号：`⌘`（Cmd）、`⇧`（Shift）、`⌥`（Alt/Option）
Windows/Linux 使用文字：`Ctrl`、`Shift`、`Alt`

```typescript
function formatKeybinding(binding: Keybinding): string {
  const isMac = navigator.platform.toUpperCase().includes("MAC");
  const parts: string[] = [];
  if (binding.mod) parts.push(isMac ? "⌘" : "Ctrl");
  if (binding.shift) parts.push(isMac ? "⇧" : "Shift");
  if (binding.alt) parts.push(isMac ? "⌥" : "Alt");
  parts.push(formatKeyName(binding.key));
  return parts.join(isMac ? " " : "+");
}
```

## 四、快捷键持久化

### 存储方案

使用现有 settings 数据库，key 为 `keybindings`，value 为 JSON 字符串。仅存储**用户修改过的快捷键**（diff），未修改的 action 使用默认值：

```json
{
  "newConnection": { "mod": true, "shift": true, "alt": false, "key": "n" },
  "toggleSidebar": { "mod": true, "shift": false, "alt": true, "key": "s" }
}
```

**加载逻辑**（settingsStore）：
1. 从数据库读取 `keybindings` JSON
2. 与 `DEFAULT_KEYBINDINGS` 合并（用户设置覆盖默认值）
3. 存入 `keybindings: ref<KeybindingMap>()`

**保存逻辑**：
1. 对比当前值与 `DEFAULT_KEYBINDINGS`，仅提取差异项
2. 序列化为 JSON 字符串
3. `settings_set("keybindings", json)`

### settingsStore 扩展

```typescript
// 新增 state
const keybindings = ref<KeybindingMap>({ ...DEFAULT_KEYBINDINGS });

// loadAll() 中新增 case
case "keybindings": {
  const custom = JSON.parse(value) as Partial<KeybindingMap>;
  keybindings.value = { ...DEFAULT_KEYBINDINGS, ...custom };
  break;
}

// 新增 action
function updateKeybinding(action: KeybindingAction, binding: Keybinding): void;
function resetKeybinding(action: KeybindingAction): void;
function resetAllKeybindings(): void;
function isKeybindingConflict(action: KeybindingAction, binding: Keybinding): KeybindingAction | null;
```

## 五、useShortcuts 重构

### 当前问题

`useShortcuts.ts` 中快捷键硬编码为 `if (mod && e.key === "n")` 条件分支。

### 重构方案

改为数据驱动：从 `settingsStore.keybindings` 读取映射，动态匹配 KeyboardEvent。

```typescript
export function useShortcuts(handlers: ShortcutHandlers) {
  const settingsStore = useSettingsStore();
  const sessionStore = useSessionStore();
  const isMac = navigator.platform.toUpperCase().includes("MAC");

  function matchesBinding(e: KeyboardEvent, binding: Keybinding): boolean {
    const mod = isMac ? e.metaKey : e.ctrlKey;
    return (
      mod === binding.mod &&
      e.shiftKey === binding.shift &&
      e.altKey === binding.alt &&
      e.key.toLowerCase() === binding.key.toLowerCase()
    );
  }

  function onKeydown(e: KeyboardEvent) {
    const bindings = settingsStore.keybindings;

    if (matchesBinding(e, bindings.toggleSidebar)) {
      e.preventDefault();
      handlers.toggleSidebar();
      return;
    }

    if (matchesBinding(e, bindings.newConnection)) {
      e.preventDefault();
      handlers.openNewConnection();
      return;
    }

    // ... 同理处理其他 action

    // goToTab1~9
    for (let i = 1; i <= 9; i++) {
      const action = `goToTab${i}` as KeybindingAction;
      if (matchesBinding(e, bindings[action])) {
        e.preventDefault();
        const tab = sessionStore.tabs[i - 1];
        if (tab) sessionStore.setActive(tab.sessionId);
        return;
      }
    }
  }

  onMounted(() => window.addEventListener("keydown", onKeydown));
  onUnmounted(() => window.removeEventListener("keydown", onKeydown));
}
```

**关键变化**：
- 所有 `if (mod && e.key === "x")` 替换为 `matchesBinding(e, bindings.xxx)`
- ShortcutHandlers 接口保持不变（action 层无变化）
- 快捷键变更后无需重新注册 listener（每次 keydown 动态读取 store）

## 六、goToTab 合并显示

原有 `goToTab`（Cmd+1~9）拆分为 9 个独立 action 以支持独立绑定，但 UI 中需合并展示以避免列表过长。

### 显示规则

- 若 `goToTab1` ~ `goToTab9` 的快捷键模式一致（仅 key 不同，且 key 分别为 1~9），合并显示为 "Go to Tab N" — `⌘ 1~9`
- 若任一被单独修改，展开为 9 行独立显示

### 实现

```typescript
const isGoToTabDefault = computed(() => {
  const bindings = settingsStore.keybindings;
  return Array.from({ length: 9 }, (_, i) => {
    const action = `goToTab${i + 1}` as KeybindingAction;
    const b = bindings[action];
    const d = DEFAULT_KEYBINDINGS[action];
    return b.mod === d.mod && b.shift === d.shift && b.alt === d.alt && b.key === d.key;
  }).every(Boolean);
});
```
