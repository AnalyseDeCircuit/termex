# V0.19.0 — SFTP Per-Tab 多开 + 灵活布局

## Context

当前 SFTP 面板是全局单实例，所有 SSH tab 共享一个 SFTP 面板。切换 tab 需要重新同步 SFTP 连接，无法同时操作多个服务器的文件。

本迭代将 SFTP 改为 **per-tab 实例**，每个 SSH 连接拥有独立的 SFTP 状态，并支持灵活的布局模式（互斥 tab / 左右分屏 / 上下分屏），通过拖拽切换。

### 当前状态

- SFTP 面板是全局单实例（`sftpStore` 管理全局 `leftPane/rightPane`）
- 切换 SSH tab 时通过 `syncToActiveSession()` 重新同步右窗格
- SFTP 面板固定在底部，通过全局 resize handle 调整高度
- 面板内部有 Files/Transfers 子 tab

### 设计决策（用户确认）

- SFTP per-tab：每个 SSH 连接独立 SFTP 实例
- 拖出后 `[SSH]` tab 消失，Terminal 独占一个区域，`[SFTP]+[Transfers]` 作为分屏面板 tab
- **统一布局**：所有 tab 共享同一布局模式（互斥/左右/上下），切换 tab 时布局不变
- **Transfers 范围**：仅显示当前 tab 的传输任务
- **SFTP 连接时机**：延迟建立，用户点击 [SFTP] tab 或拖出时才建立 SFTP channel

## 一、架构变更

### 1.1 从全局到 Per-Tab

**当前**：
```
App.vue
  └─ SftpPanel (全局单实例, v-if="sftpStore.panelVisible")
     └─ sftpStore: 全局 leftPane/rightPane/transfers
```

**新**：
```
App.vue
  └─ TerminalPane (per-tab, v-for)
     └─ TabWorkspace (新组件, per-tab)
        ├─ TerminalView (xterm.js, 从 TerminalPane 提取)
        ├─ SftpPanel (per-tab 实例, props 驱动)
        ├─ TransfersPanel (per-tab, 筛选当前 tab)
        └─ 内部 tab 栏: [SSH] [SFTP] [Transfers]
```

### 1.2 三种布局模式

```
模式 1: 互斥 Tab（默认）
┌─────────────────────────────────┐
│  [SSH] [SFTP] [Transfers]       │  ← 内部 tab 栏
├─────────────────────────────────┤
│  Terminal 或 SFTP 或 Transfers   │  ← 同一区域，互斥切换
└─────────────────────────────────┘

模式 2: 左右分屏（拖 [SFTP] 到右侧）
┌────────────────┬────────────────┐
│                │[SFTP][Transfers]│
│   Terminal     │                │
│                │   SFTP 内容     │
└────────────────┴────────────────┘

模式 3: 上下分屏（拖 [SFTP] 到下方）
┌─────────────────────────────────┐
│           Terminal               │
├─────────────────────────────────┤
│  [SFTP] [Transfers]             │
│        SFTP 内容                 │
└─────────────────────────────────┘
```

布局模式是全局设置（`settingsStore.sftpLayout`），所有 tab 使用相同布局。

### 1.3 布局切换 — 拖拽

用户拖动 `[SFTP]` tab 按钮触发布局切换：

1. **从 tab 栏拖到右侧**：出现右侧放置区高亮 → 释放 → 切换到左右分屏
2. **从 tab 栏拖到下方**：出现底部放置区高亮 → 释放 → 切换到上下分屏
3. **分屏模式下拖回 tab 栏**：释放 → 恢复互斥 tab 模式

拖拽使用 mousedown/mousemove/mouseup（Tauri WKWebView 不支持 HTML5 drag-drop）。
拖拽过程中显示放置区高亮提示（半透明蓝色区域）。

## 二、数据模型

### 2.1 Per-Tab SFTP 状态 — useTabSftp composable

每个 tab 通过 `useTabSftp(sessionId)` composable 持有独立的 SFTP 状态：

```typescript
export function useTabSftp(sessionId: Ref<string>) {
  const connected = ref(false);
  const leftPane = ref<PaneState>({ mode: "local", ... });
  const rightPane = ref<PaneState>({ mode: "remote", ... });
  const transfers = ref<TransferItem[]>([]);
  const activeSubTab = ref<"ssh" | "sftp" | "transfers">("ssh");

  async function openSftp(): Promise<void>;
  async function closeSftp(): Promise<void>;
  // ... 文件操作方法从 sftpStore 迁移
}
```

### 2.2 全局布局状态

```typescript
// settingsStore.ts
const sftpLayout = ref<"tabs" | "right" | "bottom">("tabs");
```

持久化到 settings KV 表，重启后恢复。

### 2.3 全局 sftpStore 精简

- 删除：`panelVisible`, `leftPane`, `rightPane`, `transfers`
- 保留：全局 `clipboard`（跨 tab 复制粘贴）
- 保留：SFTP channel 管理封装（`sftp_open`/`sftp_close`）

## 三、组件架构

### 3.1 TabWorkspace.vue（新增）

每个 SSH tab 的工作区容器，管理 Terminal、SFTP、Transfers 的布局和拖拽。

- Props: `sessionId`, `layout`
- 内部使用 `useTabSftp(sessionId)` 管理 SFTP 状态
- `TerminalView` 使用 `v-show` 而非 `v-if`（避免 xterm 重新初始化）
- 分屏模式下有可拖拽的 resize handle
- **SFTP 自动连接策略**：
  - tabs 模式：延迟建立，用户点击 [SFTP] tab 时才连接
  - right/bottom 分屏模式：SSH 连接成功后自动建立 SFTP channel（因为 SFTP 面板直接可见）
  - 通过 `watch(isConnected)` 监听 SSH 状态变化触发
  - SFTP 未连接时显示加载状态（"Connecting..." 脉冲动画）或 "SFTP not connected" 提示
- **SFTP 生命周期隔离**：
  - 每个 tab 的 SFTP 通过 `useTabSftp` composable 独立管理
  - 关闭 tab 时由 Rust 后端 `ssh_disconnect` 自动清理对应的 SFTP channel
  - 不再通过全局 `sftpStore.close()` 关闭（避免误杀其他 tab 的 SFTP）

### 3.2 TerminalView.vue（新增，从 TerminalPane 提取）

纯 xterm.js 容器，原 TerminalPane 的核心逻辑：
- `useTerminal` composable
- 搜索栏集成
- 关键词高亮
- 连接状态覆盖层

### 3.3 TerminalPane.vue（重构）

简化为 TabWorkspace 外壳，传递 sessionId 和 layout。

### 3.4 SftpPanel.vue（改造）

- 移除内部 Files/Transfers sub-tab（由 TabWorkspace tab 栏管理）
- 改为 props 驱动（接收 `sessionId`, `leftPane`, `rightPane`）
- 保留双窗格文件浏览功能

### 3.5 TransfersPanel.vue（改造）

- 接收 `transfers` prop 而非读全局 store
- 仅显示当前 tab 的传输任务

## 四、SFTP 终端路径同步

### 4.1 功能说明

SFTP 远程窗格工具栏新增**同步按钮**（循环箭头图标），开启后 SFTP 文件浏览器自动跟随终端的当前工作目录（CWD）。

用户在 SSH 终端中执行 `cd /var/log`，SFTP 远程窗格自动导航到 `/var/log`。

### 4.2 实现架构

```
终端中用户 cd /some/path
    ↓
后台每 3 秒通过 exec_command("pwd") 获取 CWD
    ↓
CWD 变化 → SFTP 远程窗格自动 listPaneDir(newPath)
```

**exec_command** — SshSession 新方法，通过独立 exec channel 执行命令：
- 不干扰 PTY shell（独立 channel）
- 返回 `(stdout, exit_code)`
- 同时也是 v0.20.0 tmux/Git 功能的基础设施

**Tauri 命令**: `ssh_exec(sessionId, command)` → `{ stdout, exitCode }`

### 4.3 同步按钮

- 位置：SFTP 远程窗格工具栏，New Folder 按钮右侧
- 图标：循环箭头 SVG
- 状态：
  - 未激活：灰色（默认）
  - 激活：绿色，每 3 秒轮询 CWD
- 点击切换开启/关闭

### 4.4 涉及文件

| 文件 | 变更 |
|------|------|
| `src-tauri/src/ssh/session.rs` | 新增 `exec_command()` 方法 |
| `src-tauri/src/commands/ssh.rs` | 新增 `ssh_exec` Tauri 命令 |
| `src-tauri/src/lib.rs` | 注册 `ssh_exec` 命令 |
| `src/components/sftp/SftpFilePane.vue` | 同步按钮 + CWD 轮询逻辑 |

## 五、拖拽实现

### 5.1 放置区检测

```
mousedown on [SFTP] tab button
  → start drag (threshold: 5px movement)

mousemove:
  → calculate mouse position relative to workspace
  → right 1/3 area → dropTarget = "right"
  → bottom 1/3 area → dropTarget = "bottom"
  → neither → dropTarget = null

mouseup:
  → if dropTarget → update sftpLayout
  → cleanup
```

### 5.2 放置区指示器

拖拽过程中，workspace 区域覆盖半透明高亮层：
- 右侧 1/3：蓝色半透明 + 左边框
- 底部 1/3：蓝色半透明 + 上边框
- 鼠标悬停的区域加深高亮

### 5.3 分屏 resize handle

- 左右分屏：竖向分割线（1px），拖拽调整 Terminal:SFTP 宽度比
- 上下分屏：横向分割线（1px），拖拽调整 Terminal:SFTP 高度比
- 比例存储在 `localStorage`
