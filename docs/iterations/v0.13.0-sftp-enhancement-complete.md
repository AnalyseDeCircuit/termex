# Termex v0.13.0 — SFTP 面板完整增强迭代

## 迭代概述

v0.13.0 修复 SFTP 面板的三个关键问题，并通过完整的右键菜单系统替换现有 hover 按钮交互，提供生产级文件管理体验。

- ✅ **Bug 1**: 文件列表无法滚动（Flex 容器布局修复）
- ✅ **Bug 2**: SFTP 面板底部遮挡状态栏（Overflow 溢出修复）
- ✅ **Feature**: 完整右键菜单系统（主菜单 7 项 + 更多子菜单 7 项）
- ✅ **Feature**: 剪贴板操作（复制/剪切/粘贴/移动）
- ✅ **Feature**: 文件权限修改（chmod）
- ✅ **Feature**: 文件信息查看对话框

---

## 背景与痛点

当前 SFTP 面板存在以下问题：

### 问题 1: 文件列表无法滚动

**现象**: 目录内文件数量超过面板高度时，下方文件无法滚动查看

**根本原因**: `SftpPanel.vue` 内容容器 `<div class="flex-1 min-h-0">` 是 flex item，但不是 flex container。其内的 files tab div 用了 `flex-1` 期望撑满，但父容器不是 flex，`flex-1` 无效，导致 RemoteFilePane / LocalFilePane 的滚动容器 `flex-1 overflow-auto` 因为没有确定高度而无法滚动。

### 问题 2: SFTP 面板遮挡状态栏

**现象**: 打开 SFTP 面板后，底部状态栏被面板内容覆盖

**根本原因**: `App.vue` 中 content column `<div class="flex-1 flex flex-col min-w-0">` 缺少 `overflow-hidden`。当 SFTP 面板高度（默认 256px）+其他区域撑满内容列后，内容可能从 flex 容器溢出，视觉上覆盖状态栏。

### 问题 3: 交互体验不完整

**现象**: 当前仅有 hover 按钮（下载/重命名/删除），缺少以下常用操作：
- 复制文件路径（便于终端粘贴）
- 新建文件/文件夹
- 权限修改
- 文件信息查看
- 复制/剪切/粘贴（文件级操作）

**期望**: 右键菜单系统，主菜单 + 更多子菜单，覆盖所有常用文件操作

## 核心设计

### 1. Flex 布局修复

#### SftpPanel.vue（Line 65, 67）

**修改前**:
```html
<!-- 内容容器：是 flex item，但不是 flex container -->
<div class="flex-1 min-h-0 flex flex-col">
  <!-- Files tab: 期望 flex-1 撑满，但父容器不是 flex，flex-1 失效 -->
  <div v-if="activeTab === 'files'" class="flex-1 flex min-h-0">
```

**修改后**:
```html
<!-- 内容容器：变为 flex container，建立 flex context -->
<div class="flex-1 min-h-0 flex flex-col">
  <!-- Files tab: 用 h-full 继承高度，确保撑满 -->
  <div v-if="activeTab === 'files'" class="h-full flex min-h-0">
```

**原理**:
- `flex-1 min-h-0 flex flex-col` 建立新的 flex context
- `h-full` 在此 context 下继承 100% 高度
- LocalFilePane / RemoteFilePane 内的 `overflow-auto` 因有确定高度而生效

#### App.vue（Line 258）

**修改前**:
```html
<div class="flex-1 flex flex-col min-w-0">
```

**修改后**:
```html
<div class="flex-1 flex flex-col min-w-0 overflow-hidden">
```

**原理**: `overflow-hidden` 防止子元素（包括 SFTP 面板）从 flex 容器溢出，确保 StatusBar 不被覆盖。

##### paste()
```typescript
async function paste(): Promise<void> {
  if (!sessionId.value || !clipboard.value) return

  const destPath = buildFullPath(clipboard.value.name)

  if (clipboard.value.op === 'cut') {
    // 使用 sftp_rename 实现原地移动
    await tauriInvoke("sftp_rename", {
      sessionId: sessionId.value,
      oldPath: clipboard.value.sourcePath,
      newPath: destPath,
    })
    clipboard.value = null
  } else if (clipboard.value.op === 'copy') {
    // 小文件：read + write
    const data = await tauriInvoke<number[]>("sftp_read_file", {
      sessionId: sessionId.value,
      path: clipboard.value.sourcePath,
    })
    await tauriInvoke("sftp_write_file", {
      sessionId: sessionId.value,
      path: destPath,
      data,
    })
    // 注意：copy 后保留 clipboard，支持多次粘贴
  }

  await listDir(currentPath.value)
}
```

##### createFile(name: string)
```typescript
async function createFile(name: string): Promise<void> {
  if (!sessionId.value) return
  const path = buildFullPath(name)
  await tauriInvoke("sftp_write_file", {
    sessionId: sessionId.value,
    path,
    data: [],  // 空文件
  })
  await listDir(currentPath.value)
}
```

##### chmod(entry: FileEntry, mode: u32)

### 4. 后端 chmod 命令

#### session.rs（新增方法）

```rust
/// Changes file permissions (chmod).
pub async fn chmod(&self, path: &str, mode: u32) -> Result<(), SftpError> {
    use russh_sftp::protocol::{FileAttrs, FilePermissions};

    let perms = FilePermissions::from_mask(mode);
    let attrs = FileAttrs {
        permissions: Some(perms),
        ..Default::default()
    };
    self.sftp.setstat(path, attrs).await?;
    Ok(())
}
```

#### commands.rs（新增命令）

```rust
/// Changes file permissions (chmod).
#[tauri::command]
pub async fn sftp_chmod(
    state: State<'_, AppState>,
    session_id: String,
    path: String,
    mode: u32,  // octal mask, e.g. 0o755
) -> Result<(), String> {
    let sessions = state.sftp_sessions.read().await;
    let sftp = sessions
        .get(&session_id)
        .ok_or_else(|| SftpError::SessionNotFound(session_id).to_string())?;
    sftp.chmod(&path, mode).await.map_err(|e| e.to_string())
}
```

#### lib.rs（命令注册）

在 `tauri::Builder::default().invoke_handler()` 中添加：
```rust

    // ... 其他命令 ...
    sftp_chmod,
])
```

### 5. FileInfoDialog.vue（新组件）

显示文件详细信息的模态对话框：

```vue
<script setup lang="ts">
import { ref } from "vue"
import type { FileEntry } from "@/types/sftp"
import { useI18n } from "vue-i18n"

const { t } = useI18n()

defineProps<{
  visible: boolean
  entry: FileEntry | null
}>()

defineEmits<{
  close: []
}>()

function formatSize(bytes: number): string {
  if (bytes === 0) return "0 B"
  const k = 1024
  const sizes = ["B", "KB", "MB", "GB"]
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  return Math.round((bytes / Math.pow(k, i)) * 100) / 100 + " " + sizes[i]
}

function formatDate(timestamp: number | undefined): string {
  if (!timestamp) return "—"
  return new Date(timestamp * 1000).toLocaleString()
}
</script>

<template>
  <el-dialog
    :model-value="visible"
    :title="t('sftp.fileInfo')"
    width="400px"
    @close="$emit('close')"
  >
    <div v-if="entry" class="space-y-3">
      <div class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.name') }}:</span>
        <span style="color: var(--tm-text-primary)">{{ entry.name }}</span>
      </div>
      <div class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.type') }}:</span>
        <span style="color: var(--tm-text-primary)">
          {{ entry.isDir ? t('sftp.directory') : t('sftp.file') }}
        </span>
      </div>
      <div v-if="!entry.isDir" class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.size') }}:</span>
        <span style="color: var(--tm-text-primary)">{{ formatSize(entry.size) }}</span>
      </div>
      <div class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.permissions') }}:</span>
        <span style="color: var(--tm-text-primary); font-family: monospace">
          {{ entry.permissions || "—" }}
        </span>
      </div>
      <div v-if="entry.uid !== undefined" class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.uid') }}:</span>
        <span style="color: var(--tm-text-primary)">{{ entry.uid }}</span>
      </div>
      <div v-if="entry.gid !== undefined" class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.gid') }}:</span>
        <span style="color: var(--tm-text-primary)">{{ entry.gid }}</span>
      </div>
      <div v-if="entry.mtime !== undefined" class="flex justify-between">
        <span style="color: var(--tm-text-secondary)">{{ t('sftp.modified') }}:</span>
        <span style="color: var(--tm-text-primary)">{{ formatDate(entry.mtime) }}</span>
      </div>
    </div>
  </el-dialog>
</template>
```

```typescript
function buildFullPath(name: string): string {
  if (sftpStore.currentPath === "/") {
    return `/${name}`
  }
  return `${sftpStore.currentPath}/${name}`
}
```

## 附录 C: ChmodDialog.vue 组件

一个轻量级的权限编辑对话框，支持八进制输入（e.g. 755）：

```vue
<script setup lang="ts">
import { ref } from "vue"
import type { FileEntry } from "@/types/sftp"

const props = defineProps<{
  visible: boolean
  entry: FileEntry | null
}>()

const emit = defineEmits<{
  confirm: [mode: u32]
  close: []
}>()

const modeInput = ref("")

function handleConfirm() {
  const mode = parseInt(modeInput.value, 8)
  if (!isNaN(mode)) {
    emit("confirm", mode)
    modeInput.value = ""
  }
}
</script>

<template>
  <el-dialog
    :model-value="visible"
    title="Edit Permissions"
    width="400px"
    @close="$emit('close')"
  >
    <el-input
      v-model="modeInput"
      placeholder="e.g. 755"
      style="margin-bottom: 1rem"
    />
    <template #footer>
      <el-button @click="$emit('close')">Cancel</el-button>
      <el-button type="primary" @click="handleConfirm">Confirm</el-button>
    </template>
  </el-dialog>
</template>
```

Similar pattern for NewFileDialog and MkdirDialog.
