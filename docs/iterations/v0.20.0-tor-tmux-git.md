# V0.20.0 — Tor 代理 + tmux 持久会话 + Git 远程同步

## Context

构建"数字游民"远程开发工作流：开发者从全球任意位置，通过匿名/静态住宅代理链连接远程 IDC 服务器运行 Codex 等 AI 编码工具，断开 SSH 后任务继续执行，重连后自动恢复现场，代码通过 Git 在远程与本地间同步。

### 当前状态

- v0.18.0 已支持 SOCKS5/SOCKS4/HTTP/HTTPS 代理，支持代理+跳板机组合链路
- SSH 断开后远程进程终止（无持久会话支持）
- 无远程 Git 操作能力

### 设计决策（用户确认）

- Tor 方案：**外部 Tor + 自动检测**（轻量，不嵌入 arti）
- 代码同步：**基于 Git 的远程操作**（非实时文件同步）
- 远程开发深度：**轻量方案**（代理链 + 持久会话 + 文件同步）
- tmux 是持久会话的关键：断开后任务继续运行，重连后 `tmux attach` 恢复

## 一、Tor 代理集成

### 1.1 方案说明

Tor 对外暴露标准 SOCKS5 接口（默认 `127.0.0.1:9050`），连接逻辑直接复用已有的 `connect_socks5()` 实现。用户也可以手动配置 SOCKS5 代理指向 Tor，但原生 Tor 类型提供更好的体验：

- 自动检测本地 Tor 服务状态
- 选择 Tor 类型时自动填充默认地址和端口
- 专属图标区分

### 1.2 Tor 自动检测

| 端口 | 来源 | 说明 |
|------|------|------|
| 9050 | 系统 Tor 服务 | `tor` daemon 默认监听端口 |
| 9150 | Tor Browser | Tor Browser Bundle 内置代理 |

检测方式：TCP connect 探测，1 秒超时。

```rust
pub async fn tor_detect() -> Result<TorStatus, String> {
    // 尝试连接 127.0.0.1:9050（系统 Tor）
    // 失败则尝试 127.0.0.1:9150（Tor Browser）
    // 返回 { running: bool, port: u16 }
}
```

跨平台兼容：TCP 探测在 macOS/Windows/Linux 行为一致，无需平台特定代码。

### 1.3 连接架构

```
Tor 代理连接:   TcpStream → Tor SOCKS5 (127.0.0.1:9050) → [Tor Network] → Exit Node → SSH Target
组合链路:       TcpStream → Tor SOCKS5 → [Tor Network] → Bastion SSH → direct-tcpip → Target SSH
```

### 1.4 Tor 局限性（重要提示）

| 特性 | 说明 |
|------|------|
| 延迟 | 300-500ms+，交互式 SSH 体验一般 |
| 出口 IP | 随时变化，无法保持静态 IP |
| 封锁 | 部分云服务商/GitHub 封锁 Tor 出口节点 |
| 适用场景 | 匿名需求 > 稳定性需求时使用 |

**推荐搭配**：Tor 作为可选匿名层，主通道使用静态住宅代理（SOCKS5/HTTP）保证稳定性。

### 1.5 UI

- ProxyTree 新增 Tor 类型，紫色洋葱形 SVG 图标
- 添加代理对话框选择 Tor 时：
  - 自动填充 Host: `127.0.0.1`, Port: `9050`
  - 触发 `tor_detect` 显示检测状态（绿色 = 运行中 / 红色 = 未检测到）
  - 不显示 TLS 选项（Tor 自带加密）

## 二、tmux 持久会话

### 2.1 核心问题

| 场景 | 无 tmux | 有 tmux |
|------|---------|---------|
| SSH 断开 | 远程进程被 SIGHUP 杀死 | 进程在 tmux session 中继续运行 |
| SSH 重连 | 丢失所有上下文 | `tmux attach` 恢复完整终端状态 |
| Codex 任务 | 任务中断，需要重新开始 | 任务继续执行，重连后看到结果 |

### 2.2 设计方案

**tmux 模式**（per-server 设置，存储在 servers 表）：

| 模式 | 行为 |
|------|------|
| `disabled` | 默认，普通 shell（当前行为不变） |
| `auto` | 检测远程 tmux → 可用则自动启用，不可用则降级为普通 shell |
| `always` | 始终启用 → 不可用时显示错误提示 |

**tmux session 命名**：`termex-{serverId 前 8 位}`（简短唯一）

### 2.3 连接流程

```
ssh_connect()          → SSH 认证
    ↓
ssh_open_shell()       → 打开 PTY shell
    ↓
ssh_exec("command -v tmux")  → 通过独立 exec channel 检测 tmux（不干扰 shell）
    ↓
if (tmuxMode !== "disabled" && tmux 可用):
    ssh_write("tmux new-session -A -s termex-{id8}\n")
    ↑ -A flag: session 存在则 attach，不存在则 create
    ↓
用户进入 tmux 环境，可以运行 Codex 等长时间任务
```

**断开重连**：
```
用户关闭 SSH 窗口
    ↓
tmux 进程继续运行在远程服务器
远程 Codex 任务继续执行
    ↓
用户重新连接同一服务器
    ↓
ssh_open_shell() → ssh_write("tmux new-session -A -s termex-{id8}\n")
    ↓
-A flag 检测到已有 session → 自动 attach
    ↓
用户看到之前的终端输出，Codex 任务结果完整保留
```

### 2.4 exec_command — 关键基础设施

SSH 协议支持在同一连接上打开多个 channel。`exec_command()` 通过独立的 exec channel 执行命令，不干扰用户的 PTY shell：

```rust
// session.rs
pub async fn exec_command(&self, command: &str) -> Result<(String, u32), SshError> {
    let channel = self.handle.channel_open_session().await?;
    channel.exec(true, command).await?;
    // 收集输出直到 EOF，返回 (stdout, exit_code)
}
```

此方法同时服务于：
1. tmux 可用性检测（`command -v tmux`）
2. Git 远程同步面板（`git status`, `git log`, etc.）

### 2.5 数据库变更

Migration v8：
```sql
ALTER TABLE servers ADD COLUMN tmux_mode TEXT DEFAULT 'disabled';
```

### 2.6 UI

ConnectModal → Authorization tab 中，Startup Command 下方新增：
```
tmux 模式  [Disabled ▼]
           ├─ Disabled   — 普通 shell
           ├─ Auto       — 自动检测并使用
           └─ Always     — 始终使用（需远程安装 tmux）
```

## 三、Git 远程同步面板

### 3.1 方案说明

通过 SSH exec channel 在远程服务器执行 git 命令，在本地面板显示结构化结果并提供快捷操作。这是轻量级方案——不是完整的 Git 客户端，而是日常 git 操作的快捷入口。

### 3.2 架构

```
┌─ Frontend ───────────────────────────────────────┐
│  GitPanel.vue → gitStore.ts → tauriInvoke()     │
└──────────────────┬───────────────────────────────┘
                   │ IPC
┌─ Backend ────────┴───────────────────────────────┐
│  commands/git.rs → session.exec_command()        │
│  解析 git 输出 → 返回结构化 JSON                  │
└──────────────────────────────────────────────────┘
                   │ SSH exec channel
┌─ Remote ─────────┴───────────────────────────────┐
│  cd /path && git status --porcelain -b           │
│  cd /path && git log --oneline -10               │
│  cd /path && git pull / push / commit            │
└──────────────────────────────────────────────────┘
```

### 3.3 Tauri 命令

| 命令 | 远程执行 | 返回 |
|------|----------|------|
| `git_status` | `cd {path} && git status --porcelain=v1 -b` | `GitStatus { branch, ahead, behind, changes[], isGitRepo }` |
| `git_log` | `cd {path} && git log --oneline -{count}` | `Vec<GitLogEntry { hash, message }>` |
| `git_pull` | `cd {path} && git pull 2>&1` | stdout 输出 |
| `git_push` | `cd {path} && git push 2>&1` | stdout 输出 |
| `git_commit` | `cd {path} && git add -A && git commit -m "{msg}"` | stdout 输出 |

### 3.4 Git Status 解析

`git status --porcelain=v1 -b` 输出格式：
```
## main...origin/main [ahead 2, behind 1]
 M src/main.rs
A  src/new_file.rs
D  deleted.rs
?? untracked.txt
```

- 第一行 `##`：分支名 + tracking info（ahead/behind）
- 后续行：2 字符状态码（XY）+ 空格 + 文件路径
- 状态码：`M` 修改、`A` 新增、`D` 删除、`R` 重命名、`??` 未追踪

解析逻辑作为纯函数放在 `commands/git.rs` 中，便于单元测试。

### 3.5 数据模型

```typescript
// src/types/git.ts
export interface GitStatus {
  branch: string;
  ahead: number;
  behind: number;
  changes: GitChange[];
  isGitRepo: boolean;
}

export interface GitChange {
  status: string;    // "M", "A", "D", "R", "??"
  path: string;
}

export interface GitLogEntry {
  hash: string;      // 短 hash (7 字符)
  message: string;
}
```

### 3.6 面板 UI

底部面板，与 SFTP 互斥显示（打开 Git 面板时关闭 SFTP，反之亦然），共享调整手柄。

```
┌──────────────────────────────────────────────────────────────┐
│  Git   [/home/user/project     ]  [↻]   main  ↑2 ↓0        │
├──────────────────────────────────────────────────────────────┤
│  M  src/main.rs                                              │
│  A  src/new_file.rs                                          │
│  D  old_file.rs                                              │
│  ?? untracked.txt                                            │
├──────────────────────────────────────────────────────────────┤
│  [Pull]  [Push]  [Commit: message input______  ✓]  [Sync]   │
└──────────────────────────────────────────────────────────────┘
```

- **顶部**：路径输入框（可编辑）、刷新按钮、分支名 + ahead/behind 计数
- **中部**：变更文件列表（状态图标 + 路径），compact 显示
- **底部**：操作按钮行
  - Pull：`git pull`
  - Push：`git push`
  - Commit：展开行内文本输入框 + 提交按钮
  - Sync：`git pull && git push`

快捷键：`Cmd+Shift+G`（macOS）/ `Ctrl+Shift+G`（Windows/Linux）
